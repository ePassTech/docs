---
title: "Change requests"
description: "Request amendments to approved registrations in ePass"
---

After a registration has been approved, any updates to details, premises, or registration status must go through a change request. Change requests follow a draft, review, and approval workflow so that all amendments are tracked and verified before they take effect.

No changes are applied to the system or visible on the public register until a request has been approved. This ensures that registered data remains accurate while amendments are being reviewed.

<Note>
  If you are an applicant managing your own registration through **my.ePass** (the Applicant Hub), you can update your details directly without submitting a change request. The approval workflow described on this page applies to changes made through **app.ePass** (the Admin Portal).
</Note>

## Types of change request

<Frame>
  <img src="/images/change-request-types.png" alt="Selecting a change request type from the premises page" />
</Frame>

You can submit the following types of change request from the relevant application or premises page:

| Type | Description |
| --- | --- |
| **Update details** | Change contact information, premises address, business type, or other registered details. |
| **Update status** | Set a premises to active, inactive, or suspended. |
| **Remove premises** | Archive a premises that is no longer trading or was added in error. |
| **Add premises** | Add a new premises to your approved registration. |
| **Archive registration** | Archive your entire registration, including all associated premises. |

<Warning>
  Archiving a registration affects all premises linked to that application. This cannot be undone without submitting a new application.
</Warning>

## How it works

Change requests move through four stages:

<Steps>
  <Step title="Create a draft">
    From the application or premises page, select the type of change you want to make. Fill in the required fields and provide a reason for the change. You can also upload a supporting document if needed. Your request is saved as a **draft** until you are ready to submit.
  </Step>
  <Step title="Review and submit">
    Review a summary of your proposed changes before submitting. The review page shows exactly which fields are changing, along with the old and new values. Once you submit, the request is sent to an approver for review. You will not be able to edit the request after submission.

    <Frame>
      <img src="/images/change-request-review.png" alt="Review page showing requested field changes before submission" />
    </Frame>
  </Step>
  <Step title="Approval or rejection">
    An approver in your organisation reviews the request. They can either approve it, which applies the changes immediately, or reject it with a reason. Until a request is approved, the existing data remains unchanged across the system and on the public register.
  </Step>
  <Step title="Notification">
    You receive an email when your request is approved or rejected. If rejected, the email includes the reason provided by the approver.
  </Step>
</Steps>

## One request at a time

Only one change request can be active per premises or application at a time. If a draft or pending request already exists, you will need to complete, cancel, or wait for it to be actioned before creating a new one. This prevents conflicting changes from overlapping and ensures each request is reviewed independently.

## Visibility in the system

When a change request is active, the affected premises or application is highlighted throughout the system. In premises lists, an icon and tooltip indicate that a change has been requested. On the premises detail page, a banner shows who submitted the request and when, with a link to view it.

## Creating a change request

1. Navigate to the **application** or **premises** page where you want to make a change.
2. Select the type of amendment from the available options.
3. Complete the form with the updated information.
4. Enter a reason explaining why the change is needed (required, up to 1,000 characters).
5. Optionally upload a paper form or supporting document.
6. Review the summary of your changes.
7. Click **Submit request**.

After submission, the request status changes to **requested** and an email notification is sent to approvers in your organisation.

## Cancelling a draft

If you no longer need to make a change, you can cancel a draft request at any time before submitting it. Cancelling permanently removes the draft.

## Reopening a rejected request

If your request is rejected, you do not need to start from scratch. You can reopen the rejected request, which creates a new draft pre-populated with your previous submission. Update the relevant fields, adjust your reason if needed, and resubmit.

## Approving or rejecting a request

<Info>
  You cannot approve or reject your own change requests.
</Info>

If you have approval permissions, pending change requests appear on your dashboard. To review a request:

1. Open the change request from your dashboard.
2. Review the proposed changes and the reason provided.
3. Either **approve** the request, which applies the changes immediately, or **reject** it with a reason (up to 500 characters).

The person who submitted the request receives an email notification with the outcome.

## Activity log

All change request activity is recorded in the activity log, which is visible on the premises screen. This includes when a request was created, submitted, approved, or rejected, and by whom. Where fields were changed, the log shows the old and new values. The activity log provides a full audit trail for every amendment made to a premises or application.
